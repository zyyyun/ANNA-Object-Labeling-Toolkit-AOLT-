# Feature Landscape: GS Certification Quality Requirements

**Domain:** C# WinForms Desktop Video Labeling Tool (AOLT)
**Certification Target:** GS인증 1등급 (ISO/IEC 25023 기반)
**Researched:** 2026-04-14
**Sources:** TTA GS시험·인증 서비스 소개 (공식 문서), ISO/IEC 25023:2016, KISA 보안 가이드

---

## 인증 구조 이해

GS인증 1등급은 ISO/IEC 25023, 25051, 25041 국제표준 기반으로 **9개 품질 특성 전체**를 평가한다.
8대 품질 특성(기능적합성, 성능효율성, 호환성, 사용성, 신뢰성, 보안성, 유지보수성, 이식성) + 일반적 요구사항.

**등급 구분 원칙:**
- 1등급: 모든 품질 평가 항목을 국제표준 기준에 따라 충족
- 2등급: 기본 성능·안정성은 충족하나 일부 품질 항목에서 상대적 미흡
- 결함 발견 시: 결함 리포트 송부 → 1회 보완(1등급에 한함) → 회귀 시험

**핵심 원칙:** Critical/High 등급 결함 0건이 1등급 통과의 전제조건이다.
결함 하나가 특정 부특성 전체를 불합격시킬 수 있다.

## 1. 기능적합성 (Functional Suitability)

### 부특성
- **기능 완전성**: 지정된 작업 및 사용자 목적에 맞는 기능을 제공하는 정도
- **기능 정확성**: 정확한 결과를 제공하는 정도
- **기능 적절성**: 기능이 지정된 작업 및 목적 달성을 용이하게 하는 정도

### Table Stakes (필수 — 없으면 즉시 불합격)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 모든 UI 버튼·메뉴가 오류 없이 동작 | 결함예시: "'사용자 조회' 버튼 클릭 시 오류 메시지 출력 후 기능 미실행" | Vehicle 드롭다운 교체 불가 버그 존재 — **Critical** |
| 설명서에 명시된 모든 기능이 실제로 동작 | 결함예시: "파일첨부 기능이 제공되어야 하지만 누락됨" | 설명서 미작성 상태 — 기준 없음 |
| JSON 저장·로드의 데이터 정확성 | 기능 정확성 — 잘못된 타임스탬프는 기능 오작동 | DateTime.Now 사용으로 실제 프레임 시간 아님 — **High** |
| 바운딩 박스 좌표의 정확한 계산 | 기능 정확성 — 출력 데이터 정확도 직결 | 범위 초과 좌표 클램핑 없음 — **Medium** |
| COCO 포맷 준수 JSON 출력 | 기능 완전성 — 선언된 출력 포맷 | 카테고리 ID 범위 하드코딩, 역방향 검증 없음 |

### Differentiators (차별화 — 1등급 품질 완성도 향상)

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 내보내기 전 어노테이션 유효성 검사 (너무 작은 박스, 범위 밖 좌표 경고) | 기능 적절성 — 사용자 실수 예방 | Low |
| 빈 프레임·라벨 없는 박스에 대한 경고 | 기능 완전성 — 다운스트림 ML 파이프라인 보호 | Low |
| 설명서와 실제 동작의 완전 일치 (버전 명시 포함) | 기능 완전성 — 결함예시: "프로그램과 매뉴얼에 버전이 명시되어 있지 않음" | Low |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 새 기능 추가 | GS인증은 기존 기능의 완성도 평가. 신규 기능은 추가 결함 리스크만 증가 |
| 기능 동작 방식 임의 변경 | 설명서와 불일치 발생 시 기능 적절성 결함 |

## 2. 성능효율성 (Performance Efficiency)

### 부특성
- **시간 반응성**: 응답시간, 처리시간이 요구사항을 만족하는 정도
- **자원 효율성**: 사용하는 자원의 양이 요구사항을 만족하는 정도
- **용량성**: 매개변수 최고 한도가 요구사항을 만족하는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 프레임 이동 후 UI 응답 지연 없음 (통상 3초 이내) | 결함예시: "100명 동시 투표 시 응답 30초 지연 (1명 기준 1초)" — 비율 개념 적용 | 매 Paint마다 O(n) LINQ 필터링 — **Medium** |
| 영상 로드 후 메모리 정상 반환 | 결함예시: "부하 감소 후 500MB 메모리 미반환" | FFmpeg 프로세스 미해제, 이벤트 핸들러 누수 — **High** |
| 정상적인 영상 크기(1080p 이하)에서 재생 가능 | 용량성 — 명시된 환경 내 동작 보장 | 단일 스레드 재생, 타이밍 드리프트 존재 |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 프레임별 박스 딕셔너리 인덱싱 (O(n) → O(1) 조회) | 시간 반응성 측정 가능 개선 | Low |
| 이전 비디오 로드 취소 토큰 (CancellationToken) | 자원 효율성 — 불필요한 메모리 점유 제거 | Medium |
| Undo/Redo 스택 상한 실제 적용 (MAX_UNDO_STACK=100 상수 이미 정의됨) | 자원 효율성 — 무제한 메모리 증가 방지 | Low |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 4K 60fps 지원 추가 | 현재 WinForms 단일 스레드 구조로 불가능. 새 기능이며 인증 범위 밖 |
| 스트리밍 JSON 파서 전면 교체 | 리팩토링 리스크 대비 인증 효과 미미. 현재 크기(50MB 이하)에서 미발생 |

## 3. 호환성 (Compatibility)

### 부특성
- **공존성**: 다른 소프트웨어와 환경·자원을 공유하면서 유해한 영향 없이 기능을 수행하는 정도
- **상호운용성**: 두 개 이상의 소프트웨어가 정보를 교환하고 활용하는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 백신 등 보안 SW와 충돌 없이 실행 | 결함예시: "프로그램이 백신 프로그램과 충돌하여 실행되지 않음" | 미확인 — 테스트 필요 |
| COCO JSON 출력이 타 ML 도구와 호환 | 결함예시: "연동 오류로 결제 상신 기능이 실행되지 않고 오류 메시지 출력" | 타임스탬프 오류로 JSON 정합성 깨짐 — **High** |
| 명시된 운영환경(.NET 8, Windows 10/11)에서 정상 동작 | 상호운용성 — 운영 환경 선언과 실제 일치 | 부분 확인됨 |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| JSON 로드 시 카테고리 ID 범위 검증 | 상호운용성 — 외부 시스템의 데이터 거부 방지 | Low |
| FFmpeg 미설치 시 명확한 안내 메시지 | 공존성 — 무언의 기능 실패 제거 | Low |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 크로스 플랫폼(macOS/Linux) 지원 | WinForms는 Windows 전용. GS인증도 명시된 환경만 평가 |

## 4. 사용성 (Usability)

### 부특성
- **적절 인식성**: 사용자가 제품이 자신의 니즈에 적합한지 인식하는 정도
- **학습성**: 사용자가 효과적·효율적·무위험하게 학습하는 정도
- **운영성**: 운영 및 제어를 용이하게 하는 속성
- **사용자 오류 방지성**: 사용자의 오류 발생을 방지하는 정도
- **사용자 인터페이스 심미성**: UI가 사용자와의 상호작용을 만족스럽게 하는 정도
- **접근성**: 다양한 특성과 능력을 가진 사람들이 사용할 수 있는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 도움말 제공 (툴팁 또는 인라인 도움말) | 결함예시: "프로그램에 도움말이 제공되지 않음" | 툴팁 미확인 — 추가 필요 가능 |
| 도움말의 그림/화면이 실제 UI와 일치 | 결함예시: "도움말의 그림과 프로그램 화면이 상이하여 기능 이해 어려움" | 설명서 미작성 |
| 파괴적 작업(전체 삭제 등) 전 확인 메시지 | 결함예시: "'전체 삭제' 기능 실행 시 오조작 방지를 위한 메시지가 제공되지 않음" | 미확인 — 확인 필요 |
| 모든 UI 언어 일관성 (언어 혼용 없음) | 결함예시: "프로그램 언어를 영어로 변경하였지만 일부 메뉴가 한글로 표시됨" | 전체 한국어 — 언어 혼용 없으면 해당 없음 |
| 툴바/버튼에 대한 툴팁 제공 | 사용자 인터페이스 심미성 — 기능을 알 수 없는 상태 방지 | 결함예시: "툴바에서 제공되는 기능에 대한 툴팁이 제공되지 않아 기능을 알 수 없음" |
| 오류 메시지가 구체적이고 해결 방법 제시 | 운영성 — 사용자 오류 복구 지원 | MessageBox에 generic 오류 메시지 다수 |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 상태바에 현재 작업 컨텍스트 표시 (현재 프레임, 박스 개수) | 운영성 — 사용자 인식 향상 | Low |
| Undo/Redo 가능 여부 시각적 표시 (버튼 활성/비활성) | 운영성 — 결함예시: 스택 상태 불가시 | Low |
| 저장되지 않은 변경 사항 경고 (종료 시) | 사용자 오류 방지성 — 데이터 손실 방지 직결 | Low |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 다국어(영어) 지원 추가 | 현재 인증 범위 밖. 기존 한국어 UI의 일관성 깨뜨릴 리스크 |
| 완전한 UI 리디자인 | 기존 동작 변경 없이 품질만 개선이 원칙 |

## 5. 신뢰성 (Reliability)

### 부특성
- **성숙성**: 정상 운영 상황에서 신뢰성 요구를 만족하는 정도
- **가용성**: 사용 요구 시 운영 및 접근이 가능한 정도
- **결함허용성**: 하드웨어/소프트웨어 결함이 존재하더라도 의도된 대로 동작하는 정도
- **복구성**: 중단/고장 발생 시 영향을 받은 데이터와 시스템 상태를 복구하는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 50자 이상 입력 등 경계값에서 비정상 종료 없음 | 결함예시: "50자 이상 입력 시 프로그램 비정상 종료" | 바운딩 박스 입력 유효성 검사 없음 — **High** |
| Null 참조 예외로 인한 비정상 종료 없음 | 성숙성 — 예측 가능한 입력에서 크래시 불허 | Nullable disable, NRE 리스크 다수 — **High** |
| 잘못된 SRT 파일 입력 시 크래시 없음 | 결함허용성 — 잘못된 외부 입력 처리 | SRT 파싱 실패 시 무언 실패 — **Medium** |
| 손상된 JSON 파일 로드 시 크래시 없음 | 결함허용성 — 외부 데이터 오류 처리 | generic catch 존재하나 복구 미흡 |
| 영상 로드 중 다른 파일 선택 시 상태 불일치 없음 | 성숙성 — 레이스 컨디션 | CancellationToken 없음 — **High** |
| 더블클릭 타이머 누수 없음 | 가용성 — 자원 누수로 인한 장기 실행 안정성 | 타이머 다중 생성·미해제 — **High** |
| 비정상 종료 전 데이터 저장 또는 복구 경로 제공 | 복구성 — 데이터 손실 방지 | 자동저장 없음, 백업 파일 존재하나 복구 없음 |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 시작 시 자동저장 백업 파일 감지 및 복구 제안 | 복구성 — 직접적 데이터 안전성 향상 | Medium |
| 주기적 자동저장 (예: 5분) | 복구성 — 비정상 종료 시 데이터 보호 | Medium |
| 비동기 작업 중 폼 닫기 시 안전한 종료 처리 | 결함허용성 — 비정상 상태 방지 | Medium |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 단위 테스트 프레임워크 도입 | GS인증 평가에서 테스트 코드 자체는 평가하지 않음. 리스크 대비 효과 없음 |
| MVVM 전환 | 아키텍처 리팩토링 — 기능 변경 리스크, 범위 밖 |

## 6. 보안성 (Security)

### 부특성
- **기밀성**: 인가된 사용자만 데이터에 접근 가능한 정도
- **무결성**: 인가되지 않은 접근·수정을 방지하는 정도
- **부인방지성**: 행위/이벤트가 일어났음을 부인하지 못하도록 증명하는 정도
- **책임성**: 개체의 행위를 유일하게 추적할 수 있는 정도
- **인증성**: 주체 또는 자원의 신원을 증명할 수 있는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 라이선스 인증 시 국내 비밀번호 가이드라인 수준 이상의 암호화 | 결함예시: "비밀번호 가이드라인에서 권고하지 않은 수준(예: 1111)으로 로그인 가능" | 현재 암호화 없음 — **Critical** |
| SHA-256 이상 단방향 해시 + Salt 적용 (KISA 가이드) | 프로젝트 제약사항 명시 | MAC 주소 평문 비교 방식 — **Critical** |
| 중요 이벤트(시작, 종료, 저장, 라이선스 오류)에 대한 감사 로그 | 결함예시: "중요 이벤트가 감사 로그에 기록되지 않음" | 로그 시스템 없음 (Debug.WriteLine만) — **High** |
| 접근 로그에 개인정보 미저장 | 결함예시: "사용자 접근 허용/차단 로그에 주민등록번호 저장됨" | 로그 없음 — 구현 시 주의 필요 |
| 파일 다운로드/로드 시 무결성 검사 | 결함예시: "서버로부터 설정 파일 다운로드 시 파일 무결성 검사를 수행하지 않음" | JSON 로드 시 스키마 검증 없음 |
| 파일 경로 트래버설 방지 | 무결성 — 경로 조작 공격 방지 | Path API 일부 사용, 정규화 경로 검사 없음 |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 파일 기반 구조화 로그 (로그 레벨, 로테이션) | 책임성 + 유지보수성 동시 충족 | Medium (Serilog 또는 NLog) |
| JSON 입력값 화이트리스트 검증 (라벨명 등) | 무결성 — 악의적 입력 방지 | Low |
| 백업 파일의 민감 데이터 보호 (사용 후 삭제 또는 암호화) | 기밀성 — 현재 평문 JSON.backup 노출 | Low |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 상용 라이선스 시스템 전면 교체 | 현재 MAC 기반 인증을 개선하되 전면 교체는 범위 밖 |
| 네트워크 통신 암호화 (TLS 등) | AOLT는 오프라인 데스크톱 앱 — 해당 없음 |
| 사용자 인증 시스템 신규 도입 | 현재 라이선스 모델 유지 — 새 기능 추가 금지 |

## 7. 유지보수성 (Maintainability)

### 부특성
- **분석성**: 의도적 변경이 다른 부분에 미치는 영향을 평가하거나, 수정이 필요한 부분을 식별할 수 있는 효과성·효율성
- **변경성**: 품질을 떨어뜨리지 않고 제품을 효과적·효율적으로 수정하는 정도
- **시험성**: 수립된 테스트 기준에 따라 소프트웨어 제품 자체에 테스트를 효과적·효율적으로 적용하는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 문제 발생 시 원인 진단 가능한 로그 제공 | 결함예시: "제품 구동 중 문제 발생 시 원인을 진단할 수 있는 정보(예: 로그)가 제공되지 않음" | Debug.WriteLine만 존재 — **High** |
| 설정 변경(예: 연결 포트) 후 정상 동작 | 결함예시: "DBMS 연결 포트 변경 후 DBMS에 정상 연결되지 않음" | AOLT는 포트 없음, 해당 항목은 파일 경로 변경 후 정상 동작으로 해석 |
| 구체적 예외 처리 (generic catch 최소화) | 분석성 — 오류 유형 식별 불가 방지 | generic Exception catch 다수 — **Medium** |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 파일 기반 로그 (날짜별 로테이션, 로그 레벨) | 분석성 직접 향상 — 현장 디버깅 가능 | Medium |
| 서비스별 구체적 예외 타입 정의 (VideoLoadException 등) | 변경성 — 오류 전파 경로 명확화 | Medium |
| 매직 넘버를 명명된 상수로 추출 | 변경성 — 수정 시 영향 범위 예측 | Low |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 단위 테스트 코드 작성 | GS인증 시험성(Testability) 부특성은 테스트 코드가 아닌 "테스트 적용 가능성"을 평가. 테스트 코드 자체는 평가 대상 아님 |
| 전면 아키텍처 리팩토링 (MVVM 전환) | 기존 코드 기반 개선만 허용 |

## 8. 이식성 (Portability)

### 부특성
- **적응성**: 다른 하드웨어, 소프트웨어, 운영환경에 효과적·효율적으로 적응하는 정도
- **설치성**: 특정 환경에 성공적으로 설치 또는 삭제될 수 있는 효과성·효율성
- **대체성**: 같은 환경·목적으로 사용되던 다른 소프트웨어를 대신하는 정도

### Table Stakes (필수)

| 항목 | 근거 | 현재 상태 |
|------|------|-----------|
| 명시된 OS(Windows 10/11)에서 정상 설치·실행 | 결함예시: "Windows 10 운영체제에서 프로그램이 설치되지 않음" | 부분 확인됨, Visual C++ 재배포 패키지 의존성 리스크 |
| 설치 제거 시 잔여 파일 없음 | 설치성 — 클린 언인스톨 | 미확인 |
| 이전 버전에서 제공된 기능이 현재 버전에서도 제공 | 대체성 — 결함예시: "이전 버전 기능이 시험 대상 제품에서 제공되지 않음" | 버전 관리 미확인 |
| 최소 권장 사양 문서화 및 해당 환경에서 동작 | 설치성 — 사용자취급설명서에 명시 필요 | 문서 없음 |

### Differentiators

| 항목 | 가치 | 구현 난이도 |
|------|------|-------------|
| 설치 프로그램에 .NET 8 Runtime 포함 여부 확인 및 미설치 시 안내 | 설치성 — 설치 실패 방지 | Low |
| OpenCvSharp4 DLL 버전 호환성 런타임 확인 | 설치성 — DLL 충돌 방지 | Low |
| 인스톨러에서 Visual C++ 재배포 패키지 자동 설치 | 적응성 — 의존성 누락 방지 | Medium |

### Anti-Features

| 금지 항목 | 이유 |
|-----------|------|
| 크로스 플랫폼 지원 (macOS/Linux) | WinForms는 Windows 전용, GS인증 평가 범위 밖 |
| .NET 버전 다운그레이드 | 기존 스택 유지 원칙 |

## 9. 일반적 요구사항 (General Requirements)

GS인증이 별도로 평가하는 문서 요구사항. **설명서 없으면 1등급 불가**.

### Table Stakes (필수)

| 항목 | 결함예시 | 현재 상태 |
|------|----------|-----------|
| 제품설명서 제공 (버전 명시) | "제품설명서가 제공되지 않음" | 없음 — **Critical** |
| 사용자취급설명서 제공 (정확·일관·이해 가능) | "최소 또는 권장 장비 사양 정보가 제공되지 않음" | 없음 — **Critical** |
| 버전 정보가 프로그램과 매뉴얼 모두에 명시 | "프로그램과 매뉴얼에 버전이 명시되어 있지 않음" | 없음 |
| 연동 제품 정보 제공 (FFmpeg, OpenCV 등) | "연동 제품 정보가 제공되지 않음" | 없음 |
| 품질 특성별 정보 제공 (지원 환경 명시) | "품질 특성별 정보가 제공되지 않음" | 없음 |

## 품질특성별 우선순위 요약

### 즉시 불합격 수준 결함 (Critical/High) — 최우선 수정

1. **기능적합성**: Vehicle 드롭다운 버그, JSON 타임스탬프 오류
2. **보안성**: 암호화 없음 (SHA-256 미적용), 감사 로그 없음
3. **신뢰성**: Null 참조 크래시 리스크, 타이머 누수, 레이스 컨디션
4. **일반적 요구사항**: 제품설명서·사용자취급설명서 완전 부재

### 1등급을 2등급과 차별화하는 항목 (Medium) — 중요

5. **사용성**: 툴팁, 파괴적 작업 확인 다이얼로그, 오류 메시지 구체화
6. **유지보수성**: 파일 로그 시스템, 구체적 예외 처리
7. **성능효율성**: 박스 조회 인덱싱, Undo 스택 상한 적용
8. **이식성**: 설치 프로그램 의존성 처리, 클린 언인스톨

### 품질 완성도 향상 (Low Priority — 차별화)

9. **기능적합성**: 내보내기 전 어노테이션 유효성 검사
10. **신뢰성**: 자동저장, 시작 시 복구 제안
11. **호환성**: JSON 스키마 검증

## Feature Dependencies

```
제품설명서/사용자취급설명서
  → 기능 완전성 측정 기준 (설명서가 있어야 일치 여부 판단 가능)
  → 사용자 오류 방지성 (도움말·설명서 일치 요구)
  → 이식성 설치성 (권장 사양 문서 필요)

로그 시스템 (파일 기반)
  → 유지보수성 분석성 충족
  → 보안성 책임성·부인방지성 충족
  → 두 특성을 단일 구현으로 동시 해결

암호화 (SHA-256 + Salt)
  → 보안성 기밀성·인증성 충족
  → 기존 MAC 주소 기반 라이선스 개선

예외 처리 구체화
  → 유지보수성 분석성
  → 신뢰성 성숙성 (크래시 방지)
```
