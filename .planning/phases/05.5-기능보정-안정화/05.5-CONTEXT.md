<domain>
## Phase Boundary

Phase 5.5는 Phase 6 문서화 이전에 잠긴 바이너리 기능을 최종 완성하는 보정 단계다. 사용자 현장 테스트에서 발견된 2가지 이슈를 해결한다:

1. Waypoint 선택 상태의 Entry/Exit 버튼 이중 기능 — 현재는 "프레임 지정"만 가능, 선택된 Waypoint로 **즉시 이동**이 필요
2. 영상 최초 로드 중 타임라인 상호작용 시 렉/크래시 — 로드 완료 전 입력 가드 필요

사용자 3번째 이슈(객체 ID 통합 변경)는 Phase 3 FUNC-08 수정으로 이미 해결 확인됨(수동 테스트 시나리오 A/B/C 모두 pass) → 범위 제외.

## Out of Scope
- Phase 3~5에서 완료된 기능의 리팩토링
- 새로운 UI/UX 추가 (예: 새 버튼, 새 단축키)
- Phase 6 문서화 작업 (별도 phase)

<decisions>
## Implementation Decisions

### USAB-09: Entry/Exit 버튼 이중 기능

**D-01:** 버튼 동작은 `selectedWaypoint` 상태로 분기:
- `selectedWaypoint == null` → 기존 동작 (현재 프레임을 Entry/Exit로 지정)
- `selectedWaypoint != null` → **이동 모드** (해당 Waypoint의 EntryFrame/ExitFrame으로 `LoadFrame()` 호출)

**D-02:** 수정 대상 메서드 (코드 스카우팅으로 확인됨):
- `btnEntry_Click` (MainForm.cs ~line 745) — `SetEntryMarker()` 호출 전 분기 추가
- `btnExit_Click` (MainForm.cs ~line 755) — `SetExitMarkerAndCreateWaypoint()` 호출 전 분기 추가

**D-03:** 구현 패턴 (양 버튼 동일):
```csharp
private void btnEntry_Click(object sender, EventArgs e)
{
    if (selectedWaypoint != null)
    {
        LoadFrame(selectedWaypoint.EntryFrame);
        return;
    }
    SetEntryMarker();
}
```

**D-04:** 우측 Waypoint 리스트뷰에서 Waypoint를 클릭하면 이미 EntryFrame으로 이동 + `selectedWaypoint` 설정됨 (listViewPersonWaypoints_Click 등). 사용자 흐름:
1. 우측 리스트에서 Waypoint 클릭 → EntryFrame으로 이동 + `selectedWaypoint` 설정됨 (기존)
2. Exit 버튼 클릭 → ExitFrame으로 이동 (새 기능)
3. Entry 버튼 클릭 → EntryFrame으로 다시 이동 (새 기능)

**D-05:** Waypoint 선택 해제 로직 (이미 존재): `selectedWaypoint = null` 설정 시점을 건드리지 않음. 기존 프레임 지정 기능이 그대로 동작.

### RELI-06: 영상 로드 중 타임라인 가드

**D-06:** 가드 조건: **영상이 로드되지 않았거나 로드 중일 때** 타임라인 이벤트 무시. 기준 필드: `_videoService.IsVideoLoaded`.

**D-07:** 가드 적용 위치 (3곳):
- `panelTimeline_MouseDown` (MainForm.cs ~line 2097) — 클릭 시작
- `panelTimeline_MouseMove` (MainForm.cs ~line 2110) — 드래그
- `panelTimeline_MouseUp` (MainForm.cs ~line 2116) — 드래그 종료 (isTimelineDragging 리셋은 유지)

**D-08:** 알림 없음 (사용자 요청) — 가드 시 조용히 return, MessageBox 표시 금지.

**D-09:** Person/Vehicle 수동 추적 경합 방지:
- `LoadFrame` 내부에는 `_videoService.IsVideoLoaded` 체크가 이미 존재 (line 329)
- 추가 가드: 타임라인 드래그 중 프레임 전환이 폭주하는 경우 rate limiting 검토 → 단, MVP는 "로드 완료 전 입력 차단"만 구현
- USAB-08 수동 추적은 `isDragging + LoadFrame` 경로를 공유하므로 타임라인 가드로 간접 보호됨

**D-10:** 영상 로드 상태 정의: `VideoService.LoadVideoAsync()` 완료 후 `IsVideoLoaded = true`가 되는 시점까지가 "로드 중". 이후는 "로드 완료". 기존 필드 재사용, 새 플래그 추가 안 함.

### Claude's Discretion
- 버튼 이중 기능 시 사용자 피드백 (커서 변화, 툴팁 텍스트 갱신 등) — 필요하다고 판단되면 추가
- RELI-06 구현에서 early-return 로그 레벨 (Debug vs 없음)
- 가드 조건에 `selectedWaypoint` 상태를 연계할지 여부 (현재는 로드 상태만 기준)

<specifics>
## Specific Ideas

### 버튼 동작 의미 명확화
- Waypoint 미선택 → "Entry/Exit **지정**" (기존)
- Waypoint 선택 → "Entry/Exit **이동**" (신규)
- 동일 버튼에 두 기능이 공존하지만 사용자 맥락이 명확해서 혼란 없음 (Waypoint 선택 여부는 우측 패널에서 시각 확인 가능)

### 테스트 시나리오 (UAT)
1. **USAB-09 A**: Waypoint 리스트에서 첫 항목 클릭 → Entry 버튼 → Entry 프레임으로 이동, Exit 버튼 → Exit 프레임으로 이동
2. **USAB-09 B**: Waypoint 선택 해제 후 (Escape 또는 다른 영역 클릭) → Entry 버튼 → 현재 프레임이 Entry로 지정됨 (기존 동작)
3. **USAB-09 C**: Waypoint 선택 상태에서 bbox를 클릭해서 선택 (selectedBox 설정) → Entry/Exit 버튼은 여전히 이동 모드로 동작해야 하는가? → **D-01에 따라 selectedWaypoint만 확인하므로 이동 모드**
4. **RELI-06 A**: 영상 파일 선택 → 영상 로드 표시 중 즉시 타임라인 클릭 → 클릭 무시됨, 렉/크래시 없음
5. **RELI-06 B**: 영상 로드 완료 후 → 타임라인 클릭/드래그 정상 동작 (회귀 없음)
6. **RELI-06 C**: Person/Vehicle 수동 추적 중 + 새 영상 로드 시도 → USAB-08 경로와 충돌 없음

</specifics>
